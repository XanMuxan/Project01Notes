
# Введение

В этой записке описано, как настроить и запустить LLM локально с использованием Ollama и OpenWebUI на Windows, Linux, macOS - без необходимости в Docker.

## Предварительные требования

Перед началом убедитесь, что у вас установлены следующие компоненты:

- **Python**: Необходим для работы Open WebUI.

Убедитесь, что вы используете **Python** **3.11.X**. Например, версия 3.11.6 работает хорошо. Последняя версия Python, 3.12.7, не совместима с текущей версией Open WebUI.

## Установка Ollama

1. Перейдите на [страницу загрузки Ollama](https://ollama.com/download).
2. Выберите вашу платформу (Windows, Linux или macOS) и скачайте соответствующую версию.
3. Установите Ollama, следуя стандартному процессу установки для вашей платформы.

## Установка и запуск Open WebUI

Для установки Open WebUI можно использовать менеджер пакетов Python pip. Следуйте этим шагам:

1. Откройте терминал и выполните следующую команду для установки Open WebUI:

```bash
pip install open-webui
```

2. После установки обновите pip до последней версии, выполнив:

```bash
python.exe -m pip install --upgrade pip
```

3. После установки Open WebUI запустите сервер с помощью следующей команды:

```bash
open-webui serve
```

4. После успешного запуска Open WebUI вы можете получить к нему доступ в вашем браузере по адресу: http://localhost:8080.

Кроме того OpenWebUI можно склонировать с официального репозитория на GitHub.

## Настройка Ollama и загрузка языковых моделей

По умолчанию Ollama использует следующие пути для хранения языковых моделей (LLM) на вашем устройстве:

- **macOS**: `~/.ollama/models`
- **Linux**: `/usr/share/ollama/.ollama/models`
- **Windows**: `C:\Users\%username%\.ollama\models`

Вы можете изменить этот путь. Например, на Windows используйте следующую команду:

```bash
setx OLLAMA_MODELS "D:\ollama_models"
```

### Настройка переменных среды на macOS

Если вы используете Ollama как приложение на macOS, переменные среды необходимо задавать с помощью команды `launchctl`. Для установки переменной используйте следующую команду:

```bash
launchctl setenv OLLAMA_MODELS "/new/path/to/models"
```

После установки переменной перезапустите приложение Ollama, чтобы изменения вступили в силу.

### Настройка переменных среды на Linux

Для пользователей Linux, работающих с Ollama как с сервисом systemd, переменные среды можно настроить с помощью `systemctl`. Вот как это сделать:

1. Откройте конфигурацию сервиса, выполнив команду:

```bash
sudo systemctl edit ollama.service
```

2. В разделе [Service] добавьте следующую строку:

```bash
Environment="OLLAMA_MODELS=/new/path/to/models"
```

3. Сохраните файл, затем перезагрузите systemd и перезапустите сервис:

```bash
sudo systemctl daemon-reload
sudo systemctl restart ollama
```

### Загрузка и запуск языковых моделей в Ollama

Для загрузки и запуска языковых моделей в Ollama используйте следующие команды в терминале. Эти команды автоматически загрузят модель, если она еще не установлена:

- Для загрузки и запуска модели LLaMA 3.1 от Meta (Facebook):

```bash
ollama run llama3.1
```

- Для загрузки и запуска модели Gemma 2 от Google:

```bash
ollama run gemma2
```

Так же можно запустить любую модель которая доступна на официальном сайте Ollama https://ollama.com/search:

```bash
ollama run название_модели
```


## Использование Open WebUI

После запуска Open WebUI вы можете получить к нему доступ по адресу http://localhost:8080. Перейдя по этому адресу, вы будете приглашены зарегистрироваться или войти в систему. Чтобы создать новый аккаунт, выберите опцию "Sign Up". Этот аккаунт создается локально, то есть он доступен только на этом устройстве, и никакой другой пользователь не сможет получить доступ к вашим данным.
